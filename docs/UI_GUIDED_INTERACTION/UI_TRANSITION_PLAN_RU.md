# План перехода к UI-управлению ботом

## Цель

Перейти от управления через текстовые команды к более удобному интерфейсу с меню, кнопками и пошаговыми сценариями, не ломая текущий рабочий бот и не переписывая сразу весь проект.

Команды пока сохраняются как резервный и продвинутый режим.

## Этап 1. Базовый Guided UI

1. Добавить главное меню после `/start`.
2. Вынести основные действия в кнопки:
   - `Добавить`
   - `Поиск`
   - `Список`
   - `Темы`
   - `Коллекции`
   - `Импорт/Экспорт`
   - `Бэкапы`
   - `Статистика`
3. Для частых сценариев перейти на inline-кнопки и callback-навигацию вместо ручного ввода команд.
4. Для каждого сценария сделать кнопку `Назад` и `Отмена`.
5. Убрать ручной ввод UUID там, где можно дать выбор из списка кнопками.

## Этап 2. Пошаговые сценарии

1. `Добавить запись`:
   пользователь выбирает тип `URL` или `Заметка`, потом вводит данные по шагам, затем выбирает тему кнопкой.
2. `Поиск и список`:
   сначала выбор фильтров кнопками, потом показ результатов с пагинацией.
3. `Карточка записи`:
   из результата поиска открывается экран записи с кнопками `Изменить статус`, `Открыть тему`, `Назад`.
4. `Темы`:
   просмотр дерева тем, создание и переименование через guided flow.
5. `Коллекции`:
   список коллекций, запуск, создание через форму по шагам.
6. `Бэкапы`:
   кнопки `Создать backup`, `Показать backups`, позже осторожно добавить restore-flow.

## Этап 3. Архитектурная подготовка

1. Вынести UI-логику клавиатур в отдельный модуль, например `bot/ui/`.
2. Вынести callback-data схемы в отдельный файл, чтобы не размазывать строки по handlers.
3. Переиспользовать существующие сервисы, меняя только слой входа в сценарии.
4. Сохранить обратную совместимость: старые команды продолжают работать.

## Этап 4. UX-улучшения

1. Добавить пагинацию списков.
2. Добавить кнопки быстрых фильтров по статусам.
3. Добавить `последние действия` или `быстрый доступ`.
4. Упростить тексты ответов: меньше технических формулировок, больше понятных подсказок.

## Этап 5. Drag and Drop

1. Зафиксировать, что в обычном Telegram-чате полноценный drag-and-drop почти недоступен.
2. Если drag-and-drop действительно нужен, делать это отдельной второй фазой через `Telegram Web App`.
3. В `Web App` можно будет реализовать:
   - карточки записей
   - kanban по статусам
   - drag-and-drop между колонками
   - визуальное управление темами и коллекциями

## Рекомендуемый порядок реализации

1. Главное меню и навигация.
2. Guided flow для `/add`.
3. Guided flow для `/search`, `/list`, карточки записи и смены статуса.
4. Guided flow для тем и коллекций.
5. Backup UI.
6. Отдельно решить, нужен ли потом `Telegram Web App`.

## Definition of Done для Phase 1

1. Пользователь может работать без запоминания команд.
2. Основные операции выполняются кнопками и пошаговыми экранами.
3. UUID не нужно вводить вручную в типовых сценариях.
4. Старые команды продолжают работать.
5. Есть тесты на новые UI-flow и базовая smoke-проверка.
