# Guided UI Manual Smoke Runbook (RU)

## Цель

Короткая ручная проверка, что основные menu-first сценарии Guided UI работают после изменений в `main`.

## Область проверки

- `/start` и главное меню.
- Быстрые сценарии `Список`, `Поиск`, `Коллекции`.
- Карточка записи и смена статуса.
- Контекстный возврат (`Назад`) после просмотра записи и после смены статуса.

## Предусловия

- Бот запущен и доступен в Telegram.
- База данных доступна и содержит минимум 5-10 записей в разных статусах.
- Есть хотя бы одна коллекция и как минимум один поисковый запрос с результатами.

## Смоук-сценарий (базовый, 5-10 минут)

1. Выполнить `/start`.
2. Нажать `Список`.
3. Выбрать один из быстрых фильтров статуса.
4. Открыть запись из списка.
5. Нажать `Назад` и проверить возврат в тот же контекст списка.
6. Снова открыть запись, выполнить `Изменить статус` (например, `To Read`).
7. После подтверждения статуса нажать `Назад` и проверить, что возврат сохраняет исходный контекст.
8. Перейти в `Поиск`, выполнить запрос с результатами, открыть запись и проверить `Назад`.
9. Перейти в `Коллекции`, открыть коллекцию, открыть запись из коллекции и проверить `Назад`.
10. Нажать `В главное меню` и проверить, что меню доступно из текущего состояния.

## Критерии прохождения

- Нет зависаний FSM и нерабочих callback-кнопок.
- `Назад` возвращает в ожидаемый экран/страницу, а не в произвольное состояние.
- После смены статуса возврат остается контекстным.
- Сообщения пустых состояний читаемы и дают понятное действие дальше.

## Шаблон фиксации результата

Использовать этот шаблон для каждого прогона:

```text
Дата: YYYY-MM-DD
Окружение: local|staging|prod-like
Версия: <commit_sha_or_tag>
Оператор: <name>
Итог: PASS|FAIL
Проверено:
- /start -> list -> entry -> back: PASS|FAIL
- list -> entry -> status change -> back: PASS|FAIL
- search -> entry -> back: PASS|FAIL
- collections -> entry -> back: PASS|FAIL
Дефекты/наблюдения:
- <если есть>
```
